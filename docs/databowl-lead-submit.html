Submitting leads via HTTP POST

Leads are to be submitted by making a HTTP POST request to: 

https://jve.databowl.com/api/v1/lead

HTTPS must be used and the request must be POST otherwise the lead will not be accepted.

The request must be a valid HTTP POST request. Therefore you must make sure that the request contains appropriate HTTP headers. As a minimum you must include the following headers:

  • Content-Length
  • Content-Type (Which must equal application/x-www-form-urlencoded)
  • Host

All data is to be submitted in the request body in the application/x-www-form-urlencoded format.

Response

The API will return a response in the JSON format with the following structure: 

{
            "result": "<result>",
            "lead_id":"<lead_id>",
            "error": {
            "msg": "<error message>",
            ...<additional_fields>...
            }
        }

Result will either be "created" or "error". The "error" key in the returned JSON will only be present if the result is "error". You should store the lead_id in order to reference the lead.

A result of "created" does not indicate that the lead has been accepted or rejected only that it has passed an initial set of checks and has been submitted for further processing.

Errors

If the lead has a result of "error" then error message will be one of the following:

Message Description
CONDITION_SET_REJECTED The lead you submitted has been rejected due to the rule group configuration of the campaign
DAILY_CAP_REACHED Either the daily cap on the campaign or your supplier-specific daily cap for the campaign has been reached for the day. It will not be possible to submit further leads until tomorrow or if the cap is manually raised.
DAILY_CAP_REACHED_QUARANTINED Either the daily cap on the campaign or your supplier-specific daily cap for the campaign has been reached for the day. The lead has not currently been accepted but may be accepted at a later date.
DUPLICATE_LEAD The lead you submitted has been identified as a duplicate and, therefore, has been rejected.
INVALID_CID The cid (campaign_id) supplied in the request was invalid.
INVALID_DATA_FOR_SUPPLIER_TYPE You supplied a UID for a supplier that is not an affiliate, or an SID for a supplier that is an affiliate..
INVALID_OPTIN_EMAIL You submitted an invalid email optin value
INVALID_OPTIN_EMAIL_TIMESTAMP You submitted an invalid email optin timestamp
INVALID_OPTIN_PHONE You submitted an invalid phone optin value
INVALID_OPTIN_PHONE_TIMESTAMP You submitted an invalid phone optin timestamp
INVALID_OPTIN_POSTAL You submitted an invalid postal optin value
INVALID_OPTIN_POSTAL_TIMESTAMP You submitted an invalid postal optin timestamp
INVALID_OPTIN_SMS You submitted an invalid sms optin value
INVALID_OPTIN_SMS_TIMESTAMP You submitted an invalid sms optin timestamp
INVALID_SID The sid (supplier_id) supplied in the request was invalid.
INVALID_SSID The ssid (sub_supplier_id) supplied in the request was invalid.
INVALID_TERMS_URL The terms URL you submitted is not a valid URL
INVALID_TERMS_URL_TIMESTAMP You submitted a terms URL timestamp, but you have not submitted a terms URL
MATCHED_SUPPRESSION The lead you submitted has matched a suppression file and, therefore, has been rejected
MISSING_CID The cid (campaign_id) was missing from the request. Note, if you are receiving this message but think you are already including the cid then please make sure you are submitting data as set out in this documentation.
MISSING_FIELD A required field is missing from the request.
MISSING_SID The sid (supplier_id) was missing from the request.
MISSING_TERMS_URL You submitted a terms URL timestamp, but you have not submitted a terms URL
MISSING_TERMS_URL_TIMESTAMP You submitted a terms URL, but you have not submitted a terms URL timestamp
MISSING_UID The uid (user_id) was missing from the request.
OPTIN_EMAIL_TIMESTAMP_IN_FUTURE You submitted an email optin timestamp that is in the future
OPTIN_PHONE_TIMESTAMP_IN_FUTURE You submitted a phone optin timestamp that is in the future
OPTIN_POSTAL_TIMESTAMP_IN_FUTURE You submitted a postal optin timestamp that is in the future
OPTIN_SMS_TIMESTAMP_IN_FUTURE You submitted a sms optin timestamp that is in the future
OUTSIDE_INCLUSION The lead you submitted has failed to match an inclusion file and, therefore, has been rejected
REJECTED_BY_AFFILIATES The user you submitted has been rejected by the configured affiliates rules on the campaign (:affiliateRejectionMessage)
SUB_SUPPLIER_NOT_VALID_FOR_SUPPLIER You are not able to submit leads for the sub supplier as they are not an assigned sub supplier of the specified supplier.
SUPPLIER_NOT_VALID_FOR_CAMPAIGN You are not able to submit leads for the campaign you specified in the request because you are not an assigned supplier for that campaign.
SUPPLIER_PAUSED You are currently paused on the campaign you are attempting to supply the lead for. The lead has not currently been accepted but may be accepted at a later date.
TOTAL_CAP_REACHED Either the total cap on the campaign or your supplier-specific total cap for the campaign has been reached. Therefore no further leads will be accepted until this limit is manually raised.
TOTAL_CAP_REACHED_QUARANTINED Either the total cap on the campaign or your supplier-specific total cap for the campaign has been reached. The lead has not currently been accepted but may be accepted at a later date.
